## 2. AI 模型选型与配置

### 2.1 音频特征提取模型

| 模型 | 参数量 | 用途 | 推荐理由 |
|------|--------|------|----------|
| **BEATs** (Microsoft) | ~90M | 自监督音频编码 | 动物声学表现优于 MFCC/HuBERT |
| **NatureLM-audio** (Earth Species Project) | 8B | 生物声学描述 | 专门针对动物声音训练 |
| **librosa 经典特征** | 无 | 补充特征向量 | 极低成本，提供有效补充信息 |

**实测性能参考**（来自 naturelm-cats 项目）：
- BEATs + 统计池化（mean+std+max）→ 5-fold CV 准确率 81.5%
- 混合 BEATs + 经典特征 → 82.2%
- MLP 分类器 1 层 128 隐藏单元 + dropout 0.3

### 2.2 情境分类器设计

```python
# 分类器架构
输入层: BEATs 768-dim 特征向量 + 统计池化 (mean, std, max) → 2304-dim
隐藏层: Dense(128) → BatchNorm → ReLU → Dropout(0.3)
输出层: Dense(N) → Softmax  # N = 情境类别数

# 猫情境类别（初始）
MEOW_CLASSES = [
    "玩耍兴奋", "饥饿要求", "焦虑不安",
    "领地警戒", "求关注", "发情",
    "疼痛不适", "友好问候", "愤怒攻击"
]

# 狗情境类别（初始）
BARK_CLASSES = [
    "兴奋玩耍", "饥饿要求", "分离焦虑",
    "领地警戒", "陌生人警告", "恐惧",
    "疼痛不适", "友好问候", "无聊"
]
```

### 2.3 LLM 选型方案

| 方案 | 优势 | 劣势 | 成本 |
|------|------|------|------|
| **Claude API** | 指令遵循强、反幻觉好 | 按 token 付费 | ~¥0.01-0.02/次 |
| **GPT-4o / Gemini 2.5** | 多模态支持 | 成本略高 | ~¥0.01-0.03/次 |
| **本地开源模型** (Qwen2.5 7B) | 零 API 成本，隐私好 | 需 GPU，效果略差 | 一次性硬件成本 |
| **混合方案** | 默认云 API + 高并发降级为本地 | 架构复杂 | 折中 |

**推荐方案：云端 API 为主，本地模型为降级备用。**

### 2.4 LLM Prompt 设计

**System Prompt（核心指令）：**

```markdown
你是一个宠物叫声分析助手。你的工作是将结构化的声学分析
数据转化为易懂的自然语言解读。

核心原则：
1. 诚实 — 置信度低于 0.6 时必须标注"不确定"
2. 具体 — 引用具体声学特征作为判断依据
3. 不拟人化过度 — 不要编造宠物"想说"的话
4. 输出严格的 JSON 格式

输出 JSON Schema:
{
  "emotion": "主要情绪标签",
  "intent": "可能的意图",
  "confidence": 0.0-1.0,
  "confidence_label": "高/中/低",
  "reasoning": "判断依据（1-2句话，引用声学特征）",
  "suggestion": "给主人的行动建议",
  "uncertainties": ["不确定的因素1", "不确定的因素2"]
}
```

## 3. 数据流详解

### 3.1 翻译请求完整流程

```
1. 用户点击"录音"按钮
2. 客户端请求麦克风权限 → 开始录音
3. 实时 VAD 检测，自动或手动停止
4. 本地音频预处理：
   a. 降噪 (spectral gating)
   b. 重采样到 16kHz
   c. 分段 (3-5秒最佳片段)
5. 发送 POST /api/v1/translate
   Body: {audio: base64, pet_id: "xxx", context: "playing"}
6. 后端：
   a. BEATs 编码 → 768-dim 特征向量
   b. 统计池化 → 2304-dim
   c. MLP 分类器 → 情境概率分布
   d. 组装结构化输入 → LLM 推理
   e. LLM 返回 JSON 结果
7. 后端返回 Response：
   {
     "id": "trans_xxx",
     "emotion": "兴奋",
     "intent": "邀请玩耍",
     "confidence": 0.86,
     "confidence_label": "高",
     "reasoning": "叫声短促有节奏，频率偏高，符合猫在逗猫棒诱导下的兴奋叫声特征",
     "suggestion": "可以继续使用逗猫棒互动，猫目前情绪积极",
     "uncertainties": [],
     "processing_time_ms": 1200
   }
8. App 展示结果（动画 + 卡片布局）
```

### 3.2 反馈循环数据流

```
用户看到结果 → 点赞或点踩
                  ↓
           POST /api/v1/feedback
           {trans_id, is_correct, correct_label?, note?}
                  ↓
       反馈入数据库 → 定期触发模型微调
                  ↓
       模型更新 → 下次翻译更准确
```

## 6. 扩展性设计

### 6.1 多物种扩展

```python
# 分类器工厂模式 — 按物种加载不同分类头
class SpeciesClassifier:
    def __init__(self):
        self.backbone = BEATsEncoder()  # 共享基础编码器
        self.classifiers = {
            "cat": CatMLPHead(),
            "dog": DogMLPHead(),
            "rabbit": RabbitMLPHead(),  # Phase 2
            "bird": BirdMLPHead(),      # Phase 2
        }

    def predict(self, audio, species):
        features = self.backbone.encode(audio)
        return self.classifiers[species].predict(features)
```

- BEATs 基础编码器跨物种共享（无需重新训练）
- 每个物种只需要训练一个轻量 MLP 分类头（~200K 参数）
- 新增物种只需 500-1000 条标注数据即可上线

### 6.2 个性化适配

```python
class PetPersonalizer:
    """
    每只宠物独立维护一个轻量级适配层。
    基于用户反馈数据进行少量梯度更新。
    """
    def __init__(self, base_model, pet_id):
        self.base = base_model  # 冻结共享参数
        self.adapter = LoRALayer(dim=128, rank=8)  # 仅训练 adapter
        self.pet_id = pet_id
        self.feedback_buffer = []  # 暂存最近反馈

    def update(self, feature, feedback):
        self.feedback_buffer.append((feature, feedback))
        if len(self.feedback_buffer) >= 10:  # 累积 10 条反馈后微调
            self._finetune()
            self.feedback_buffer.clear()
```
